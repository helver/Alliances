Vector Data Integrity Suite

The Vector Data Integrity Suite is a series of TCL applets that operate under a single GUI interface. The current set of operations available include "Connection Tracing", "Billing Reconciliation", and "UPC Contract Tagging Validation".

In order to use this tool you need the following pieces:

Expect - Install this first.
vector_tag.exp - Grab this.
vector_connections.exp - And grab this.

In order to run the program, you need to have access to the G: drive. Without the G: drive, you won't have any switches available to you, so you won't get much done.

The set up involves adding the path to "wish80" to your %PATH%. How to do that is beyond the scope of this document. Additionally, the two files "vector_tag.exp" and "vector_connections.exp" should be in the same directory.

Assuming that you put the two "vector_" programs in the directory called "C:\vector", you would run the program by going into a dos prompt and issuing the following commands:

      cd c:\vector
      wish80 vector_tag.exp

This will start the program up in GUI mode - which is the most useful. However the program can also be run in command-line mode. The syntax is as follows:

      tclsh80 vector_tag.exp [tcb] 127.0.0.1

In this case, the options [tcb] indicate that you can run the program with between 1 and 3 of the options given. The 127.0.0.1 would be replaced by the IP address of the Vector switch that you wanted to check up on. The output will come back on standard out (to the screen) but you can redirect the output to a file like so:

      tclsh80 vector_tag.exp [tcb] 127.0.0.1 > c:\output1.txt
